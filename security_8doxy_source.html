5 # Security # {#sec_main}
6 
7 Two key aspects of securing an application are _authentication_ and _authorization_.
8 While __authentication__ is the process of verifying the identity of a user,
9 __authorization__ means determining if the user is allowed to do what he or she
10 is about to do. That implies that authentication is a precondition for
11 authorization.
12 
13 ## Users and Roles ## {#sec_users_roles}
14 
15 Users are essential for an authentication system. In wCMF users are represented
16 as instances of classes implementing the
17 \link wcmf::lib::security::principal::User `User`\endlink interface. Since wCMF's
18 authorization system is role based, users are organized in roles. Role classes
19 implement the \link wcmf::lib::security::principal::Role `Role`\endlink interface.
20 
21 ## Authentication ## {#sec_authentication}
22 
23 In wCMF authentication is handled by implementations of
24 \link wcmf::lib::security::AuthenticationManager `AuthenticationManager`\endlink.
25 
26 Currently the framework provides
27 \link wcmf::lib::security::impl::DefaultAuthenticationManager `DefaultAuthenticationManager`\endlink,
28 which implements a __login/password__ based authentication procedure by matching
29 the given user credentials against existing
30 \link wcmf::lib::security::principal::User `User`\endlink instances. These instances
31 are provided by implementations of
32 \link wcmf::lib::security::principal::PrincipalFactory `PrincipalFactory`\endlink.
33 
34 The following code demonstrates the authentication process as implemented
35 in \link wcmf::application::controller::LoginController `LoginController`\endlink:
36 
37 ~~~~~~~~~~~~~{.php}
38 // get the user credentials from the request
39 $login = $request->getValue('user');
40 $password = $request->getValue('password');
41 
42 try {
43  // try to login using the credentials
44  $authManager = ObjectFactory::getInstance('authenticationManager');
45  $authUser = $authManager->login($login, $password);
46 
47  // set the authenticated user in the session
48  $session = ObjectFactory::getInstance('session');
49  $session->setAuthUser($authUser);
50 }
51 catch (Exception $ex) {
52  Log::error("Could not log in: ".$ex, __CLASS__);
53 }
54 ~~~~~~~~~~~~~
55 
56 The configuration of the authentication process in the default application looks
57 like the following:
58 
59 ~~~~~~~~~~~~~{.ini}
60 [AuthenticationManager]
61 __class = wcmf\lib\security\impl\DefaultAuthenticationManager
62 principalFactory = $principalFactory
63 
64 [PrincipalFactory]
65 __class = wcmf\lib\security\principal\impl\DefaultPrincipalFactory
66 userType = app.src.model.wcmf.User
67 roleType = app.src.model.wcmf.Role
68 ~~~~~~~~~~~~~
69 
70 Since
71 \link wcmf::lib::security::principal::impl::DefaultPrincipalFactory `DefaultPrincipalFactory`\endlink
72 retrieves user instances from the storage, it needs to configured with the
73 appropriate entity types. If required, the default user type may be replaced
74 by custom implementations of \link wcmf::lib::security::principal::User `User`\endlink.
75 
76 @note The authenticated user, that is associated with the current session is
77 obtained using the \link wcmf::lib::core::Session::getAuthUser `Session::getAuthUser`\endlink
78 method.
79 
80 ## Authorization ## {#sec_authorization}
81 
82 The purpose of authorization is controlling access to application resources,
83 which could be controllers or entity instances. To establish __access control__,
84 rules have to defined in the first place an enforced afterwards.
85 
86 ### Permissions ### {#sec_perm}
87 
88 Access control rules are expressed as _permissions_. Permissions are either
89 granted or denied to a role (see \ref sec_users_roles). A permission in wCMF
90 is a combination of a _resource_, a _context_ and an _action_ and the notation
91 is similar to action keys (see \ref arch_actionkey).
92 
93 A resource could be one of the following:
94 
95 - Controller class name (e.g. `wcmf\application\controller\SaveController`)
96 - Type name (e.g. `app.src.model.wcmf.User`)
97 - Type and propery name (e.g. `app.src.model.wcmf.User.login`)
98 - Object id (e.g. `app.src.model.wcmf.User:123`)
99 - Object id and propery name (e.g. `app.src.model.wcmf.User:123.login`)
