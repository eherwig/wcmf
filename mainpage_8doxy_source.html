10 <h2>Model-Driven Software Development</h2>
11 wCMF strongly encourages Model-Driven Software Development (<em>MDSD</em>) to let you focus on the application domain and
12 increase productivity significantly. Creating a wCMF-based application basically involves three steps:
13 
14 <div class="row">
15  <div class="col-md-4"><h3><i class="fa fa-sitemap fa-2x pull-left"></i> Model</h3>
16  Start by modeling your application in wCMF's UML based Domain Specific Language (<em>DSL</em>).
17  It not only supports defining the domain model, controllers and views, but also configuration and application flow.
18  Graphical editors are available from the <a href="http://www.eclipse.org/papyrus/" target="_blank">Eclipse</a> and
19  <a href="http://sourceforge.net/projects/olympos/" target="_blank">Olympos</a> projects.
20  </div>
21  <div class="col-md-4"><h3><i class="fa fa-gears fa-2x pull-left"></i> Build</h3>
22  Run the code generator to create persistence mappers, controllers, views and configuration files from the model.
23  After that a web application based on the <a href="http://dojotoolkit.org/" target="_blank">Dojo Toolkit</a> is ready to be used.
24  It allows to manage entity instances and includes a full text search, localization support and user/role management.
25  </div>
26  <div class="col-md-4"><h3><i class="fa fa-expand fa-2x pull-left"></i> Extend</h3>
27  wCMF's clear object oriented design is based on well known design patterns, that allow to extend the application in every aspect.
28  Custom code is protected from subsequent generator runs, which ensures that future requirements can be implemented
29  while model and code stay in synch.
30  </div>
31 </div>

33 <h2>Features</h2>
34 \endhtmlonly
35 
36 - Full featured object persistence layer:
37  - Flexible mapper architecture with adapter to RDBMS
38  - Optimistic and pessimistic object locking
39  - Searching using template based object query and criteria api
40  - Query caching and eager relation loading
41  - Transaction support
42 - Presentation layer based on the <a href="http://www.smarty.net/" target="_blank">Smarty Template Engine</a>
43 - Configuration of the application flow through config files
44 - Dependency injection support
45 - Basic event processing
46 - Role based permission management (for actions, types, instances, instance properties)
47 - <a href="http://framework.zend.com/manual/1.12/en/zend.search.lucene.overview.html" target="_blank">Lucene</a> search engine integration
48 - I18n support
49 - Flexible logging (<a href="http://logging.apache.org/log4php/" target="_blank">Apache log4php</a>)
50 - SOAP and REST interface supporting CRUD operations on all objects
51 - Code generator for model-driven software development
52 - Available under an open source license (MIT)
