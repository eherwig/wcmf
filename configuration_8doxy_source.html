5 # Configuration # {#conf_main}
6 
7 Application configuration is handled by implementations of
8 \link wcmf::lib::config::Configuration `Configuration`\endlink. To use the configuration
9 througout the application, an instance of the appropriate implementation is
10 passed to \link wcmf::lib::core::ObjectFactory `ObjectFactory`\endlink. This class
11 provides \ref conf_di and access to the configuration.
12 
13 ~~~~~~~~~~~~~{.php}
14 $configPath = '../config/';
15 $configFile = 'config.ini';
16 
17 // configure ObjectFactory
18 $config = new InifileConfiguration($configPath);
19 $config->addConfiguration($configFile);
20 ObjectFactory::configure($config);
21 
22 // get a pre-configured instance
23 $session = ObjectFactory::getInstance('session');
24 
25 // get a configuration value
26 $config = ObjectFactory::getConfigurationInstance();
27 $tz = $config->getValue('timezone', 'application');
28 ~~~~~~~~~~~~~

30 ## Configuration Format ## {#conf_format}
31 
32 Currently wCMF provides an implementation for using the
33 [INI file](http://en.wikipedia.org/wiki/INI_file) format.
34 \link wcmf::lib::config::impl::InifileConfiguration `InifileConfiguration`\endlink
35 handles ini files that consist of **key-value pairs** that are grouped in **sections**.
36 
37 ~~~~~~~~~~~~~{.ini}
38 [SectionA]
39 key1 = value1
40 key2 = value2
41 
42 [SectionB]
43 key3 = value3
44 ~~~~~~~~~~~~~
45 
46 Additionally it defines **array values** (enclosed in curly braces) and **inclusion**
47 of additional files. Included files are defined in the `include` key of the `Config`
48 section. Keys in the current file override keys from included files.
49 
50 The following example shows how to include an array of additional configuration
51 files in order to separate concerns:
52 
53 ~~~~~~~~~~~~~{.ini}
54 [Config]
55 include = {server.ini, persistence.ini, presentation.ini, security.ini}
56 ~~~~~~~~~~~~~
57 
58 @note While path values in the configuration are normally resolved relative to the
59 value of the `WCMF_BASE` constant, the included files are expected to be located
60 in the directory that is passed to the constructor of
61 \link wcmf::lib::config::impl::InifileConfiguration `InifileConfiguration`\endlink.

63 ## Dependency injection ## {#conf_di}
64 
65 wCMF supports [Dependency injection](http://en.wikipedia.org/wiki/Dependency_injection)
66 through the \link wcmf::lib::core::ObjectFactory `ObjectFactory`\endlink class.
67 This class implements _setter injection_, which means that dependencies are either
68 set through a _setter method_ of the client class or a _public member variable_.
69 
70 Assembly details for client instances are defined in the application configuration
71 in special sections containing a `__class` key. The value of the key denotes the
72 class to be used to create the client instance. All other keys are mapped to
73 setters/members of that class. Complex dependencies of an instance may be set by
74 using the `$` notation, which refers to another configuration section. If a section
75 does not contain a `__class` key it is instantiated as an associative array.
76 
77 The following example demonstrates the pattern by means of the
78 \link wcmf::lib::persistence::PersistenceFacade `PersistenceFacade`\endlink instance:
79 
80 ~~~~~~~~~~~~~{.ini}
81 [PersistenceFacade]
82 __class = wcmf\lib\persistence\impl\DefaultPersistenceFacade
83 mappers = $typeMapping
84 logging = false
85 logStrategy = $auditingLogStragegy
86 
87 [AuditingLogStragegy]
88 __class = wcmf\lib\persistence\output\impl\AuditingOutputStrategy
89 
90 [TypeMapping]
91 app.src.model.wcmf.DBSequence = $app_src_model_wcmf_DBSequenceRDBMapper
92 app.src.model.wcmf.Lock = $app_src_model_wcmf_LockRDBMapper
93 
94 [app_src_model_wcmf_DBSequenceRDBMapper]
95 __class = app\src\model\wcmf\DBSequenceRDBMapper
96 connectionParams = $database
97 
98 [app_src_model_wcmf_LockRDBMapper]
99 __class = app\src\model\wcmf\LockRDBMapper
100 connectionParams = $database
101 
102 [Database]
103 dbType = sqlite
104 dbHostName = 127.0.0.1
105 dbName = app/test-db.sq3
106 dbUserName =
107 dbPassword =
108 dbCharSet = utf8
109 ~~~~~~~~~~~~~
110 
111 The first section defines that the instance named `PersistenceFacade` is an object of
112 the class \link wcmf::lib::persistence::impl::DefaultPersistenceFacade `DefaultPersistenceFacade`\endlink.
113 The _logging_ property is set to _false_, the _logStrategy_ property is set to an
114 instance of \link wcmf::lib::persistence::output::impl::AuditingOutputStrategy `AuditingOutputStrategy`\endlink.
115 The _TypeMapping_ section does not contain a `__class` key, which means that the
116 _mappers_ property of `PersistenceFacade` will be an associative array. In our
117 case it maps class names (e.g. _app.src.model.wcmf.DBSequence_) to mapper instances
118 (e.g. `DBSequenceRDBMapper`). Further sections show that the _connectionParams_
119 property of the `DBSequenceRDBMapper` instance is set to an associative array
120 describing the database connection (section _Database_).
121 
122 @note Configuration section and key names are treated case-insensitive.
123 
124 The `PersistenceFacade` instance may be retrieved later by using the following
125 code:
126 
127 ~~~~~~~~~~~~~{.php}
128 $persistenceFacade = ObjectFactory::getInstance('persistenceFacade');
129 ~~~~~~~~~~~~~

131 ### Shared and Non-Shared instances ### {#conf_shareddi}
132 
133 By default all instances created by \link wcmf::lib::core::ObjectFactory `ObjectFactory`\endlink
134 are _shared_ instances, which means, that they will be created on the first call
135 and only returned by succeeding calls. If you want to create a _non-shared_ instance,
136 you must use the `__shared` key in configuration and set it to _false_.
137 
138 The following example shows the configuration of a non-shared
139 \link wcmf::lib::presentation::view::View `View`\endlink instance:
140 
141 ~~~~~~~~~~~~~{.ini}
142 [View]
143 __class = wcmf\lib\presentation\view\impl\SmartyView
144 __shared = false
145 compileCheck = true
146 caching = false
147 cacheLifetime = 3600
148 cacheDir = app/cache/smarty/
149 ~~~~~~~~~~~~~

151 ## Configuration for individual users ## {#conf_peruser}
152 
153 wCMF's user base class \link wcmf::lib::security::principal::impl::AbstractUser `AbstractUser`\endlink
154 has a persistent property `config` which may be used to set a individual configuration
155 for a user. This will be loaded after the standard configuration and extends its settings.
156 If two keys have the same name the one from the user configuration overwrites the standard one.
157 Using the _RoleConfig_ configuration section, you can easily assign configuration files
158 to groups:
159 
160 ~~~~~~~~~~~~~{.ini}
161 [RoleConfig]
162 administrators = admin.ini
163 ~~~~~~~~~~~~~
164 
165 If a user is added to a listed role, the given configuration file will be automatically
166 assigned to it, if no individual configuration is set yet.
