5 # Internationalization & localization # {#i18n_main}
6 
7 _Internationalization_ of an application requires to identify all language dependent
8 resources and make them exchangeable for the actual _localization_ into a specific
9 language. This includes __static__ and __dynamic texts__ as well as __images__.
10 Since images are referenced by their filename or represented as text (e.g. _base64_
11 encoded), it is sufficient to focus on text.
12 
13 ## Static text ## {#i18n_static}
14 
15 Static text refers to text used in __application templates__ or __messages__ generated
16 while the application is executed. To keep the application language agnostic only
17 so called _text identifiers_ are used in these places and the actual translation
18 is retrieved by calling the \link wcmf::lib::i18n::Message::getText `Message::getText`\endlink
19 method:
20 
21 ~~~~~~~~~~~~~{.php}
22 $textId = 'An error occured';
23 
24 // retrieve the translation
25 $message = ObjectFactory::getInstance('Message');
26 $errorText = $message->getText($textId);
27 
28 // use the translation
29 $ex = new Exception($errorText);
30 ~~~~~~~~~~~~~
31 
32 @note Implementations of \link wcmf::lib::i18n::Message `Message`\endlink
33 must return the _text identifier_, if no translation is found or the translation
34 is empty. This helps to simplify the translation process, if the default language
35 is used in _text identifiers_. In this case, no translation is required for the
36 default language.
37 
38 If no language parameter is passed to the
39 \link wcmf::lib::i18n::Message::getText `Message::getText`\endlink method, the
40 returned translation will be in the application's __default language__. This
41 language is determined in the following way:
42 
43 1. Use the value of the `language` parameter of the `Message` configuration section
44 2. If not defined use the value of the global variable <code>$_SERVER['HTTP_ACCEPT_LANGUAGE']</code>
45 
46 If parts of the text vary depending on the application state, it is necessary to
47 put __variables__ into the text identifier. Variable names are `%0%`, `%1%`, ...
48 and they are replaced in the order of the values passed into the
49 \link wcmf::lib::i18n::Message::getText `Message::getText`\endlink method:
50 
51 ~~~~~~~~~~~~~{.php}
52 $textId = 'User %0$ is logged in since %1%';
53 $parameters = array($username, $date);
54 
55 // retrieve the translation
56 $message = ObjectFactory::getInstance('Message');
57 $loginMessage = $message->getText($textId, $parameters);
58 
59 // use the translation
60 $view->setValue('loginMessage', $loginMessage);
61 ~~~~~~~~~~~~~
62 
63 ### Providing translations ### {#i18n_static_trans}
64 
65 Since \link wcmf::lib::i18n::Message `Message`\endlink defines the interface, it
66 needs concrete implementations to actually get the translations. The
67 \link wcmf::lib::i18n::impl::FileMessage `FileMessage`\endlink class retrieves
68 translations from the file system. It expects one file per language defining
69 a language dependent array that maps text identifiers to translations.
70 The language code (e.g. _en_) is used to identify the language of the translation.
71 The file must be named like <em>messages_</em>code.<em>php</em>,
72 the variable like <em>$messages_</em>code.
73 
74 The following __examples__ show definitions for English (_en_) and German (_de_)
75 texts, where the array keys are the text identifiers used in the application and
76 the values are the translations in the appropriate language.
77 
78 <em>messages_en.php</em>
79 
80 ~~~~~~~~~~~~~{.php}
81 $messages_en = array();
82 $messages_en['%0% selected'] = '';
83 $messages_en['Author'] = '';
84 $messages_en['Book'] = '';
85 ~~~~~~~~~~~~~
86 
87 <em>messages_de.php</em>
88 
89 ~~~~~~~~~~~~~{.php}
90 $messages_de = array();
91 $messages_de['%0% selected'] = '%0% ausgewählt';
92 $messages_de['Author'] = 'Autor';
93 $messages_de['Book'] = 'Buch';
94 ~~~~~~~~~~~~~
95 
96 @note Note that the text identifiers are in the application's default language
97 (_en_) and therefor no translations need to be provided.
98 
99 The default __configuration__ of \link wcmf::lib::i18n::Message `Message`\endlink
100 is as follows:
101 
102 ~~~~~~~~~~~~~{.ini}
103 [Message]
104 __class = wcmf\lib\i18n\impl\FileMessage
105 localeDir = app/locale/
106 language = en
107 ~~~~~~~~~~~~~
108 
109 The parameter `localeDir` defines the directory where
110 \link wcmf::lib::i18n::impl::FileMessage `FileMessage`\endlink searchs for
111 translation files and the `language` parameter defines the application's default
112 language.
